# 🎬 컴포넌트/상태관리 과제

이번 과제에서는 컴포넌트 분리와 상태 관리, 스타일링 라이브러리 사용 등을 연습합니다.
`MSW`를 이용하여 Mock API 서버를 세팅해두었으므로, 실제 API와 유사한 흐름을 경험할 수 있습니다.

<hr><br>

## 📌 과제 목표

### 1. 컴포넌트 분리 연습

- 제가 미리 만들어둔 `App.tsx`에는 조회 + 추가 + 검색 기능이 모두 한 파일에 들어있습니다.
- 이를 적절히 분리하여 재사용 가능한 컴포넌트로 리팩토링하세요.
  - ex. `MovieList.tsx`, `MovieForm.tsx`, `SearchBar.tsx`, `Header.tsx` etc...
- 컴포넌트로 분리하고 해당 컴포넌트를 적절한 폴더로 분류해주세요.

### 2. 스타일링

- 현재는 기본적인 패딩, 색깔 정도만 인라인으로 들어가 있습니다.
  - 색, 테마는 제가 styles 폴더에 palette, theme으로 분리해두었습니다.
- 스타일링 라이브러리(ex. styled-components, emotion, tailwindcss 등)를 자유롭게 선택하여 원하는 대로 꾸며주세요.

### 3. 상태 관리

- 전역 상태 관리도 시도해보세요!
- 단, 모든 상태를 무작정 전역에 두지 말고, 전역으로 관리할만한 상태가 어떤 것인지를 판단해보고 그 상태들만을 전역으로 관리하는 식으로 로직을 합리적으로 설계하세요.

### 4. 커스텀 훅 설계

- 데이터 페칭, 상태 관리 등 공통 로직을 단일 책임을 가진 커스텀 훅으로 분리해주세요.
  - ex. `useMovies`, `useMovieSearch`, `useMovieForm` etc...

### 5. PR 템플릿 사용

- 해당 레포지토리에 PR 템플릿을 설정해두었습니다.
- PR 템플릿을 사용해 본인이 수행한 과제에 대해 자세히 작성해주세요.

<hr><br>

## 📂 폴더 구조

초기 제공되는 폴더 구조는 다음과 같습니다.

```typescript
├── 📂public
├── 📂src
│   ├── 📂components
│   ├── 📂hooks
│   ├── 📂mocks
│   ├── 📂styles
│   ├── 📂types
│   ├── App.tsx
│   ├── main.tsx
│   ├── vite-env.d.ts
┗── 📜etc (setting files)
```

- **mocks** → 수정 금지 (msw 세팅 완료)
- **styles** → 테마 및 색 팔레트 정의. 색 변경을 원하거나 수정이 필요할 경우 수정 가능
- **components, hooks, types** → 더미 파일만 있음. 클론 후 더미 파일을 지우고 이 폴더들에 컴포넌트를 적절히 분리해주세요.
- 필요하다면 **store, models, constants** 등 새로운 폴더를 자유롭게 추가해도 됩니다.

## 🚀 주요 기능 (구현되어있음)

### 1. 영화 목록 조회

- API(`/api/movies`)에서 영화 목록을 가져와 화면에 표시합니다.
- 로딩 중임을 화면에 표시합니다.
- API 요청 실패 시 에러 메시지를 표시합니다.

### 2. 영화 검색

- 영화 제목을 기준으로 검색어와 부분 일치하는 영화만 목록에 표시합니다.
- 검색어가 없을 경우 전체 영화 목록을 보여줍니다.

### 3. 영화 추가

- 새로운 영화를 추가할 수 있는 폼을 제공합니다.
  - 필수 입력값: 제목, 감독, 연도, 장르, 평점(0~10)
- 모든 값이 입력되지 않으면 추가되지 않고, 에러 메시지를 표시합니다.
- 성공적으로 추가되면 목록이 갱신됩니다.

### 4. 테마 변경

- 다크/라이트 모드를 전환할 수 있습니다.
- 전체 화면의 배경색, 텍스트 색, 컴포넌트 색상 등이 테마에 따라 변경됩니다.

## 🛠 주요 리팩토링 포인트
### 1. 컴포넌트 구조
- **`Header`** - 앱 제목과 테마 토글 버튼 담당
> 전역 테마 상태를 직접 사용하여 독립적으로 작동

- **`MovieForm`** - 영화 추가 폼 및 입력 검증 담당
> 폼 상태 관리를 내부에서 처리하고, 성공 시 자동 초기화

- **`MovieList`** - 영화 목록 표시 및 검색 기능 담당
> 검색 상태를 내부에서 관리하고, 로딩/빈 상태 처리

- **`MovieItem`** - 개별 영화 정보 표시 담당
> 영화 데이터만 받아서 일관된 형태로 렌더링

- **`SearchBar`** - 검색 입력 UI 담당
> 재사용 가능한 검색 컴포넌트로 설계

- **분리 이유**
> 단일 책임 원칙 준수, 재사용성 향상, 코드 가독성 개선, 유지보수성 증대


### 2. 커스텀 훅
- **분리된 훅**
> `useMovieList` - 영화 목록 조회, 로딩 상태 관리, 새로고침 기능
> `useAddMovie` - 영화 추가 API 호출 및 로딩 상태 관리
> `useMovieError` - 에러 상태 관리 및 에러 처리 로직

- **분리 근거**
> 단일 책임 원칙으로 기능별 분리, 독립적인 테스트 가능, 필요한 기능만 선택적 사용, 관심사의 명확한 분리


### 3. 전역 상태 관리
- **구현 기능**
> 다크/라이트 모드 테마 변경, localStorage를 통한 테마 설정 영구 저장

- **사용한 기술**
> Zustand - 가벼운 상태 관리 라이브러리, persist 미들웨어로 자동 저장

- **설계 이유**
> 테마는 모든 컴포넌트에서 필요하므로 전역 관리 필요, Zustand는 boilerplate가 적고 TypeScript 지원 우수, props drilling 완전 제거

### 4. 스타일링
- **사용한 기술**
> styled-components, Pretendard 웹폰트

- **개선사항**
> 인라인 스타일을 styled-components로 변환, 테마 기반 동적 스타일링, 한국어 최적화 폰트 적용, 일관된 타이포그래피 시스템 구축
